# Инструкция по публикации библиотеки

## Подготовка к публикации

1. Убедитесь, что все изменения закоммичены и запушены в репозиторий:
   ```bash
   git add .
   git commit -m "Release v1.0.0"
   git push origin main
   ```

2. Создайте тег версии:
   ```bash
   git tag -a v1.0.0 -m "Release version 1.0.0"
   git push origin v1.0.0
   ```

## Автоматическая публикация

После создания тега GitHub Actions автоматически:
- Запустит тесты
- Создаст релиз на GitHub
- Опубликует библиотеку в pkg.go.dev

## Ручная публикация

### Публикация в pkg.go.dev

Go автоматически индексирует модули из GitHub. После создания тега и релиза:
1. Дождитесь индексации (обычно несколько минут)
2. Проверьте доступность: https://pkg.go.dev/github.com/mentatxx/go-mail-ru-client

### Использование GoReleaser (опционально)

Если установлен GoReleaser:
```bash
goreleaser release --clean
```

## Проверка публикации

После публикации проверьте:
- [ ] Модуль доступен: `go get github.com/mentatxx/go-mail-ru-client`
- [ ] Документация доступна: https://pkg.go.dev/github.com/mentatxx/go-mail-ru-client
- [ ] Релиз создан на GitHub
- [ ] Все тесты проходят

## Версионирование

Следуйте [Semantic Versioning](https://semver.org/):
- **MAJOR** версия при несовместимых изменениях API
- **MINOR** версия при добавлении функциональности с обратной совместимостью
- **PATCH** версия при исправлении ошибок с обратной совместимостью
